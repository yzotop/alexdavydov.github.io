Ratio / линеаризация.

30 заметок · 9 компаний.

Материалы Компании Авторы Синтезы Карта

§ 01 В чём метод

Ratio-метрика — метрика-отношение двух сумм пользовательских сигналов: средний чек = ΣGMV / Σorders, CTR = Σclicks / Σviews, средняя длина сессии = Σtime / Σsessions. Особенность: знаменатель не равен числу пользователей — несколько заказов/показов могут принадлежать одному юзеру, наблюдения зависимы.

Стандартный t-test требует независимости — и на ratio-метриках даёт неверную оценку дисперсии. Чтобы корректно сравнить группы, есть три подхода:

  1. Бутстрап по объектам (пользователям) — семплируем юзеров с возвращением, считаем ratio в каждой подвыборке
  2. Дельта-метод — аналитическая формула дисперсии отношения через ряд Тейлора первого порядка
  3. Линеаризация — преобразуем ratio в среднюю поюзерную метрику с независимыми наблюдениями, после чего корректен обычный t-test

Формула линеаризации: L_i = X_i − k · Y_i, где k = ΣX_control / ΣY_control (global ratio по контролю). После преобразования среднее L — это и есть линеаризованная ratio-метрика, к ней применимы все методы для пользовательских метрик (включая CUPED).

§ 03 Где работает хорошо

  • Метрики бизнес-смысла — почти все ключевые продуктовые метрики (CTR, средний чек, ARPPU, конверсия в покупку при множественных событиях) ratio по своей природе
  • Когда нужна сонаправленность с целевой метрикой — линеаризация её гарантирует (направление эффекта и значимость консистентны с исходной ratio, Kuper)
  • Производственные A/B-платформы — Kuper и Avito встроили линеаризацию как стандартный шаг
  • Switchback и композитные метрики — назначаемость, удержание заказа: дельта-метод (Citymobil)
  • Big-data global metrics — Poisson/weighted bootstrap позволяют масштабировать (VK)
  • На предэкспериментальных данных — для CUPED нужна линеаризация поверх ratio с обеих периодов

§ 04 Подводные камни

  • t-test «в лоб» на ratio даёт неверный p-value. Дисперсия отношения не считается стандартными формулами для среднего — нарушается предпосылка независимости.
  • «Среднее средних» ≠ глобальный CTR. Если на пользователя посчитать ratio и усреднить — получится не то же самое, что Σclicks / Σviews. Биномиальное распределение vs честное глобальное среднее (Kuper).
  • Композитные метрики ломаются больше всего. Когда числитель и знаменатель — суммы случайных величин, обычная выборочная оценка даже среднего смещена (Citymobil: ratio estimator bias).
  • Бутстрап для ratio: семплировать по объектам, не по событиям. Иначе одного пользователя «разделим» между группами — оценка смещена (X5, Kuper).
  • CUPED не применим напрямую к ratio. Сначала линеаризовать, затем CUPED поверх — иначе CUPED работает на зависимых данных (Kuper).
  • Sample Ratio Mismatch. Если по факту в группе доли не совпадают с дизайнерскими — все ratio-метрики смещены; проверять SRM в первую очередь (expf).
  • ratio_correction при множественных тестах: эффективный MDE растёт через формулу r + 2 + 1/r (Ozon).
