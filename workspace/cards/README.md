# cards — LinkedIn carousel renderer

Офлайн-инфраструктура для создания LinkedIn-карусельных постов в формате PNG.
Каждая карточка — 1080×1350 px (4:5), рендерится через Playwright без обращения к сети.

---

## Структура

```
workspace/cards/
├── _fonts/               — .woff2 файлы (latin + cyrillic subsets)
├── _shared/
│   ├── fonts.css         — @font-face декларации
│   └── cards.css         — общие стили всех карточек
├── _template/
│   └── template.html     — пустой шаблон одной карусели
├── render.py             — рендер в PNG через Playwright
└── monetization-funnel/  — пример готовой карусели
    ├── cards.html
    └── out/              — PNG-файлы после рендера
```

---

## Как добавить новую карусель

Каждая карусель — отдельная подпапка внутри `workspace/cards/` с файлом `cards.html`.
`render.py` принимает имя этой подпапки как единственный аргумент.

1. Скопировать `_template/template.html` в новую директорию:
   ```
   cp -r _template my-carousel
   mv my-carousel/template.html my-carousel/cards.html
   ```
2. Открыть `my-carousel/cards.html`, наполнить карточки содержимым.
   Каждый `<div class="card">` → отдельный PNG-файл.
3. Из директории `workspace/cards/` запустить рендер, передав имя подпапки:
   ```
   python3 render.py my-carousel
   ```
4. PNG-файлы появятся в `my-carousel/out/card-01.png`, `card-02.png`, ...

## Рендер

```
python3 render.py <имя-директории>
```

**Требования:**
```
pip install playwright
playwright install chromium
```

Скрипт открывает `<директория>/cards.html` через `file://` URL, ждёт
загрузки всех шрифтов (`document.fonts.ready`), и снимает скриншот
каждого `.card` элемента. Выходное разрешение — 2160×2700 px (2× retina).

## Почему два шрифта вместо одного

На сайте используется Space Grotesk. У него нет кириллицы.
В браузере это незаметно — Google Fonts подставляет нужный subset динамически.
При офлайн-рендере через `file://` сетевых запросов нет, поэтому кириллица
выпадает в системный fallback.

Решение — `unicode-range` в `@font-face`. Логическое семейство `'Sans'`
состоит из двух файлов:
- **Space Grotesk** — для латиницы, чисел, символов ASCII
- **Manrope** — для кириллицы (U+0400–052F)

Браузер выбирает нужный файл автоматически, посимвольно.
В HTML и CSS достаточно писать `font-family: 'Sans'` — никаких условий.

Латинские слова в русском тексте (CPM, ARPU, D7, fill rate) отрисовываются
Space Grotesk, русский текст — Manrope. Визуально два шрифта неразличимы:
оба геометричные, с близким x-height и силуэтом.

## Почему Playwright, а не «открыть в браузере»

- Playwright рендерит пиксельно стабильно на любой машине.
- Точный `device_scale_factor=2` гарантирует retina-разрешение без зависимости от настроек экрана.
- `document.fonts.ready` ждёт полной загрузки шрифтов до снимка — без этого первая карточка иногда получает системный fallback.
- Именование файлов (`card-01.png`, `card-02.png`) и структура `out/` воспроизводимы и идемпотентны.
- Браузер с дисплейным масштабом ≠ 1× или нестандартными шрифтами даст другой результат.

---

## Почему нет Google Fonts

Google Fonts не работает при `file://` URL — CORS блокирует загрузку.
Все шрифты хранятся локально в `_fonts/` как `.woff2` subsets.
Файлы получены из `@fontsource/*` пакетов через `npm pack` и содержат
только нужные unicode-range подмножества, а не полные шрифты.
