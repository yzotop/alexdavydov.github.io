Warehouse Ops Simulator (Live) — Описание

2D, event-driven симуляция операций склада во времени: очереди, мощности (workers/servers), WIP-лимиты и окна/интервалы отгрузки перевозчика. Дискретная модель: 1 тик = 1 минута. Симуляция детерминированна по seed.

Что симулируем

Поток заказа проходит зоны:

  • INBOUND → STORAGE → PICK → PACK → OUTBOUND → CARRIER

У каждой зоны есть очередь (WIP) и ограничение по мощности (сколько заказов может обрабатываться параллельно).

Как читать “Arena”

  • Прямоугольники — зоны процесса, подпись показывает текущий WIP (queue + in-service).
  • Точки — заказы (сэмпл для скорости). Цвет = текущая зона.
  • Красное кольцо — заказ уже “late” (текущее время > SLA deadline).
  • Стрелки показывают направление потока; если WIP растёт в зоне — это симптом bottleneck.

Метрики

  • Shipped — отгружено заказовU
  • Backlog — создано минус отгружено
  • SLA breach % — доля отгруженных после дедлайна
  • Avg lead time — среднее время от создания до отгрузки
  • Utilization — занятость пулов (pick/pack) = busy / capacity

Эксперименты

  • Сделайте pickers меньше → посмотрите рост WIP в PICK и увеличение lead time.
  • Уменьшите carrier capacity → OUTBOUND станет хвостом, SLA ухудшится.
  • Задайте storage WIP cap сильно ниже → появятся блокировки “вверх по потоку”.
  • Сравните interval vs windows для перевозчика при том же объёме.
Для воспроизводимости фиксируйте seed и изменения параметров; одинаковый seed даёт одинаковую траекторию при одинаковых настройках.
