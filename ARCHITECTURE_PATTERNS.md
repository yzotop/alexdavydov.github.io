# ARCHITECTURE_PATTERNS.md
Версия: 0.1

Цель: описать архитектурные паттерны проекта для обеспечения консистентности и правильного использования Cursor.

---

## 0. Общая идея архитектуры

Проект построен как статический сайт с модульной структурой. Каждая страница — самодостаточный HTML файл с встроенными CSS и JS.

**Ключевые принципы:**
- Один файл = одна концепция
- Минимальные зависимости между страницами
- Единый визуальный стиль (см. `STYLE_GUIDE_CURSOR.md`)
- Единые дизайн-паттерны (см. `DESIGN_PATTERNS.md`)
- **Единый слой языка (`GLOSSARY.md`): определения не дублируются, используются ссылки**

## Тип: Glossary / Language layer
### Назначение
- фиксирует определения и смысл терминов проекта
- обеспечивает единый язык для курсов, калькуляторов и решений

### Примеры
- `GLOSSARY.md` (MVP)
- будущий `/lab/glossary/index.html`

### Характеристики
- один источник истины
- в контенте страниц используются ссылки на термины
- допускаются короткие tooltips, но не копирование определений
