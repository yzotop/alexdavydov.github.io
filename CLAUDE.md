# CLAUDE.md

Корневой файл инструктажа для Claude Code в этом проекте.

## Обязательно при старте каждой задачи

1. Прочитать `/working-protocol.md` — рабочий протокол
   сессии (остановки, str_replace, отчёты, pre-commit
   проверки).
2. Прочитать `/design-system.md` — конвенции дизайна
   (классы, токены, паттерны вёрстки).

Эти файлы — источник правды. Если в них что-то противоречит
запросу пользователя — остановиться и спросить, не
импровизировать.

## Ключевые жёсткие правила

### Никаких inline-стилей в HTML

Запрещено использовать атрибут `style="..."` на элементах.
Всегда либо существующий класс из `/style.css`, либо новый
класс в `/style.css`.

Единственное исключение: динамические стили, задаваемые
из JavaScript в runtime (например, position элемента,
вычисляемая ширина, данные из пользовательского ввода).
Статичные оформительские стили — всегда через классы.

См. `/design-system.md` раздел про CSS-конвенции.

### Pre-commit отчёт обязателен

Перед `git commit` всегда показывать пользователю:
- `git status` (точный список файлов)
- `git diff --stat` (масштаб)
- `git diff` по ключевым файлам (если объём <500 строк)
- Полный вывод `curl` по всем затронутым URL (HTTP-коды
  для каждого, не только сводку «всё 200»)

Без явного вывода всех проверок — не коммитить, ждать
явного «ок» от пользователя.

См. `/working-protocol.md` раздел «Процедурные требования».
