01 Зачем этот кейс

Рынок б/у MacBook на Avito — это тысячи объявлений без единой структуры. Заголовки написаны кто во что горазд: «макбук эир м1 8/256 как новый!!!» и «Apple MacBook Air 13 M1 8GB 256GB SSD Space Gray» — это один и тот же товар, но машине об этом не сказали.

Цель — собрать достаточно данных, распарсить конфигурации из свободного текста, и ответить на простые вопросы: сколько стоит конкретный сегмент (чип × размер × RAM), какие сегменты самые массовые, и — честно — не переплатил ли я за свой M4.

Побочный продукт: инфраструктура для ежедневных снапшотов, из которых можно оценить ликвидность (через исчезновение объявлений как proxy продаж).

02 Данные и сбор

Avito отдаёт максимум ~50 страниц на один поисковый запрос. При ~50 объявлениях на страницу это потолок в ~2 500 листингов — и они повторяются при переборе. Чтобы выйти за это окно, используется query plan: набор запросов, каждый с разной комбинацией параметров.

Города Москва, СПб, Новосибирск, Екатеринбург, Казань, Краснодар, Ростов + вся Россия
Сортировки По дате, релевантности, цене ↑, цене ↓ — 4 окна на каждый город
Ценовые бины 0–40k, 40–70k, 70–110k, 110–160k, 160k+ — перекрывающие сегменты
Запросы «macbook», «macbook pro», «macbook air» — разный recall из поиска

Итого: 20 запросов, суммарный target ~9 300 объявлений. После глобальной дедупликации по listing_id остаётся уникальная выборка.

Безопасность сбора

Avito агрессивно блокирует автоматизацию. Пайплайн построен так, чтобы минимизировать риск:

  • Throttle: 1–2.5 секунды между страницами, 60–180 секунд backoff при soft-block
  • Resume: checkpoint после каждой страницы. Прерывание Ctrl+C безопасно — следующий запуск продолжит с места остановки
  • Global dedup: один набор seen_ids на весь план. Объявление из Москвы не соберётся повторно из запроса «вся Россия»
  • Persistent browser state: cookies и localStorage сохраняются между сессиями
  • Soft-block detection: мало карточек, одинаковый контент, маркеры капчи → автоматический cooldown

03 Качество данных: парсинг конфигураций

Авито не отдаёт структурированные характеристики в поисковой выдаче. Единственный источник — свободный текст заголовка. Парсер извлекает пять параметров: chip, family (Air/Pro), size, RAM, SSD.

Параметр Определён Не определён Примеры паттернов

family 99.8% 1 Air, Pro, «макбук», MacBook (без Air/Pro) → MacBook
size 81% 102 13", 13.3, 13-inch, 13 дюймов, air15 → 15
RAM Извлекается из слэш-нотации (16/512) и явных «16GB», «16гб»
SSD 256GB, 512GB, 1TB; «1024GB» → 1TB

12% объявлений с неопределённым чипом — это в основном заголовки вида «Макбук в идеале» или «Apple ноутбук». Они исключаются из чип-сегментации, но участвуют в общей ценовой статистике.

06 Ликвидность (proxy продаж)

Avito не публикует данные о продажах. Но если объявление исчезло между двумя снапшотами — это proxy: либо продано, либо снято. Disappearance rate по сегментам позволяет оценить относительную ликвидность.

Честный дисклеймер: на текущий момент два снапшота (22 и 23 февраля) идентичны — 0% исчезновений. Это означает, что либо интервал слишком короткий, либо оба снапшота построены из одних и тех же сырых данных. Для валидной оценки ликвидности нужны ежедневные сборы в течение 1–2 недель.

Инфраструктура для этого готова: scripts/build_dataset.py генерирует датированные снапшоты, scripts/snapshot_compare.py автоматически находит два последних и строит отчёт с сегментацией.

07 Я купил M4 за 77k и кто я после этого

Купил MacBook на M4 за 77 000 ₽ и, как любой нормальный аналитик, вместо того чтобы радоваться покупке — собрал 547 объявлений, написал парсер на regex и построил пайплайн, чтобы проверить, не клоун ли я.

Построить целый аналитический проект, чтобы оправдать покупку ноутбука — это либо профессиональная деформация, либо прокрастинация. Я честно не уверен, что из двух.

Вердикт по данным

Медиана M4 (все конфигурации): 80 000 ₽. 77k — это 38-й перцентиль: только 38% объявлений M4 дешевле. Формально — чуть ниже медианы.

Но медиана — опасный инструмент, если не учитывать сегмент. M4 13" (Air) — медиана 73 000 ₽. M4 15" (Air) — медиана 100 245 ₽. M4 14" (Pro) — медиана 124 490 ₽.

Если мой M4 — 13-дюймовый Air, то 77k — чуть выше медианы сегмента. Нормальная цена, не великая сделка. Если 15-дюймовый Air — то 77k на 23% ниже медианы, и это уже хорошо.

Вывод: корректный verdict возможен только внутри сегмента (чип × размер × RAM × SSD). «Средняя цена M4» — это средняя температура по больнице.

08 Что можно улучшить

  • Ежедневные снапшоты. Cron + build_dataset.py + snapshot_compare.py. Через 7–14 дней — реальная оценка ликвидности по сегментам.
  • Извлечение конфигурации из карточки. Сейчас парсим только заголовок. Внутри объявления есть структурированные характеристики — это поднимет определённость с 88% до ~98%.
  • Масштаб до 10k. Query plan готов: 90 запросов × 30 городов. ✓ Реализовано. Coverage report + global dedup + 10k cap.
  • Расширение на другие SKU. Тот же пайплайн работает для iPhone, PS5, любого SKU на Avito. Нужен только новый query plan и парсер заголовков.

09 Методология и ограничения

Источник данных: публичная поисковая выдача Avito (категория «Ноутбуки»). Февраль 2026. Извлечение через Playwright (headful browser + persistent state). 547 уникальных объявлений после дедупликации по listing_id. Сбор через query plan: до 90 запросов × 30 городов × 4 сортировки.

Парсинг конфигураций: regex-парсер на Python. Поддерживает кириллицу («16гб», «М1 Про»), слэш-нотацию (16/512), дюймовые форматы (13.3", 13-inch, 13 дюймов), инференс чипа по году (≤ 2019 → Intel). Протестирован на 42 edge-case заголовках.

Ограничения:

  • Avito ≠ продажи. Объявления — это предложение, не спрос. Цена в объявлении может не совпадать с ценой сделки.
  • Disappearance rate ≠ sold rate. Объявление может исчезнуть по множеству причин: продано, снято, перезалито с новым ID, скрыто модерацией.
  • Snapshot bias. Один снапшот видит только текущие объявления. Быстро проданные товары (высокая ликвидность) недопредставлены.
  • Парсинг из title. 12% объявлений с неопределённым чипом — систематический gap для «ленивых» заголовков.
  • Региональный bias. Московские и петербургские объявления перепредставлены из-за структуры Avito.

Стек: Python, Playwright, pandas, matplotlib. Код — локальный проект. Данные: снапшот Avito, февраль 2026.
