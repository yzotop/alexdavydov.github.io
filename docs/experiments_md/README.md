# Экспорт курса "Математика экспериментов в монетизации" в Markdown

Этот каталог содержит Markdown-версии всех страниц курса из `/lab/experiments/`.

## Как экспортировать

Запустите скрипт экспорта из корня репозитория:

```bash
python3 scripts/export_experiments_md.py
```

Скрипт:
- Читает все HTML файлы из `lab/experiments/`
- Извлекает основной контент (исключая навигацию, футер, кнопки)
- Конвертирует HTML → Markdown
- Сохраняет в `docs/experiments_md/` с YAML frontmatter

## Структура файлов

Каждый `.md` файл содержит:

1. **YAML frontmatter**:
   ```yaml
   ---
   title: "Название страницы"
   source_html: "/lab/experiments/filename.html"
   ---
   ```

2. **Markdown контент** — основной текст страницы

## Проверка ссылок

После экспорта проверьте ссылки:

```bash
python3 scripts/audit_experiments_links.py
```

Скрипт проверит:
- Все ссылки на `.html` файлы в MD
- Существование соответствующих файлов в `lab/experiments/`
- Выведет список битых ссылок (если есть)

## Ручная очистка

Если в начале MD файла есть предупреждение:

```
> ⚠️ TODO: manual cleanup — main container not detected
```

Это означает, что скрипт не смог автоматически выделить основной контент. Нужно:
1. Открыть исходный HTML файл
2. Вручную удалить навигацию, футер, кнопки
3. Обновить MD файл

## Обновление контента

**Важно**: MD файлы являются source-of-truth для контента курса.

При изменении контента:
1. Редактируйте `.md` файлы в этой папке
2. HTML файлы можно обновлять вручную или через отдельный скрипт импорта (если будет создан)

## Формат ссылок

Ссылки на другие страницы курса сохраняются в формате:
- `[Текст ссылки](./role-of-experiment.html)` — относительные ссылки
- `[Текст ссылки](/lab/experiments/index.html)` — абсолютные ссылки

Все ссылки должны указывать на существующие `.html` файлы в `lab/experiments/`.
