# Предложение по улучшению структуры курса
## "A/B-тестирование: принятие решений на практике"

## 1. Краткое описание новой структуры

Курс переориентирован на **4 типа решений**, которые принимает аналитик при чтении эксперимента:

1. **Можно ли верить эффекту?** — проверка достоверности (stat sig vs practical sig, power, устойчивость, лаги)
2. **Что именно дало эффект?** — декомпозиция механизма (цена vs объём, воронка, позиция, качество)
3. **Можно ли масштабировать?** — проверка системного эффекта (сегменты, каннибализация, total-first)
4. **Что делать с неудачным экспериментом?** — работа с негативными и нулевыми результатами

**Ключевые изменения:**
- Главная страница фокусируется на 4 типах решений, а не на модулях
- Syllabus структурирован по типам решений, а не по темам
- Практика оформлена как тренажёр с группировкой по типам решений
- Каждый lesson и scenario явно привязан к типу решения
- Checklist структурирован по 4 типам решений

## 2. Маппинг сценариев на типы решений

### Решение 1: Можно ли верить эффекту?
- **003** — Статзначимо, но бизнес-эффекта нет (stat sig ≠ practical sig)
- **004** — Ничего не произошло (power/MDE, закрытие теста)
- **006** — Окно меняет вывод (лаги, устойчивость, stop rules)

**Уроки:** 02-statistics-vs-decision.html, 04-time-matters.html

### Решение 2: Что именно дало эффект?
- **001** — CPM вырос, а выручка упала (механизм: цена vs объём)
- **002** — Выручка выросла без роста CPM (механизм: объём)
- **008** — Lazy Load — рост через воронку (механизм: воронка показов)
- **009** — Рост видимости без роста давления (механизм: позиция)
- **010** — Больше показов — меньше денег (механизм: качество инвентаря)

**Уроки:** 01-price-vs-money.html

### Решение 3: Можно ли масштабировать?
- **005** — Сегменты спорят (Simpson) — проверка сегментов
- **007** — Локальная победа, системный проигрыш (каннибализация, total-first)
- **011** — Адаптивное давление по сегментам (сегментация работает)
- **012** — Вовлечённость без денег (каннибализация внимания)

**Уроки:** 03-aggregates-lie.html

### Решение 4: Что делать с неудачным экспериментом?
- **001** — CPM вырос, а выручка упала (негативный эффект)
- **004** — Ничего не произошло (нулевой эффект)
- **010** — Больше показов — меньше денег (негативный эффект)
- **012** — Вовлечённость без денег (негативный эффект)

**Уроки:** 05-real-decisions.html (частично)

## 7. Преимущества новой структуры

1. **Фокус на решениях:** Курс читается как практическое руководство, а не как справочник
2. **Логическая целостность:** Каждый элемент привязан к типу решения
3. **Понятная навигация:** Пользователь всегда знает, где он и куда идти дальше
4. **Устойчивость к расширению:** Новые сценарии легко добавляются в соответствующий тип решения
5. **Чёткое разделение:** Курс = теория, Практика = тренажёр (не дублирование)
