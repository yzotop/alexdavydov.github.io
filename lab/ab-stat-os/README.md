# Курс №5 — Статистика A/B-тестирования: операционная система аналитика

## Философия

Этот курс — не учебник по статистике. Это операционная система для принятия решений на основе экспериментов.

Академическая статистика учит формулам. Мы учим думать: какой тест выбрать, почему метрика ведёт себя так, когда результат достоверен, а когда — иллюзия.

Каждый модуль строится вокруг решения, а не вокруг теоремы.

## Отличие от академической статистики

| Академический подход | Этот курс |
|---|---|
| Начинает с формул | Начинает с продуктовой задачи |
| Доказывает теоремы | Строит decision trees |
| Предполагает нормальность | Показывает реальные распределения |
| Один правильный тест | Карта выбора с trade-offs |
| Игнорирует операционные ошибки | Посвящает им отдельный модуль |

## Связь с Курсом №4

Курс №4 («Аналитика продукта: как читать систему, а не цифры») формирует системное мышление аналитика — умение видеть продукт как систему метрик, петель обратной связи и ограничений.

Курс №5 даёт инструментарий для проверки гипотез внутри этой системы. Без Курса №4 эксперименты превращаются в изолированные тесты. Без Курса №5 системное мышление остаётся без инструмента валидации.

## Роль симуляторов

Симуляторы — не иллюстрации. Это среда для формирования интуиции.

- **Metric Classifier** — интерактивная классификация метрик перед выбором теста
- **Test Selection Map** — decision tree выбора статкритерия
- **Distribution Playground** — визуализация реальных распределений продуктовых данных
- **Cluster Simulator** — демонстрация роста дисперсии в кластерных экспериментах
- **CUPED Simulator** — снижение дисперсии через ковариатную коррекцию

## Целевая аудитория

- Senior-аналитики, которые проводят A/B-тесты и принимают решения по их результатам
- Аналитические лиды, которые ревьюят дизайн экспериментов команды
- Product-аналитики, которым нужна операционная уверенность в статистических методах

## Структура

8 модулей, от каузальности до квази-экспериментов. Каждый модуль содержит лекцию, практику и привязку к симулятору.

```
01_causality          — Что делает A/B-тест каузальным
02_metric_types       — Классификация метрик перед выбором теста
03_distributions      — Реальные распределения продуктовых данных
04_stat_tests_map     — Decision tree выбора статкритерия
05_variance_reduction — MDE, Power, CUPED, стратификация
06_cluster_tests      — Geo-тесты, кластерная рандомизация, ICC
07_common_mistakes    — Типовые ошибки аналитиков в экспериментах
08_quasi_experiments  — A/B без A/B: DiD, matching, synthetic control
```
