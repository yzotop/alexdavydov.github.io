# Практика — Типы метрик

Для каждой метрики:
- **(a)** Определите тип: Per-user / Proportion / Ratio / Conditional
- **(b)** Выберите подход к тестированию (высокоуровнево)
- **(c)** Назовите главный риск (если есть)

---

## Задачи

### 1. CTR (Click-Through Rate)
Клики / Показы. Считается глобально по всем показам. Каждый показ — либо клик, либо нет.

### 2. Конверсия в покупку (CR)
Доля пользователей, совершивших хотя бы одну покупку, от всех пользователей в группе.

### 3. ARPU (Average Revenue Per User)
Средняя выручка на пользователя. Считается как среднее revenue_per_user по всем пользователям (включая нулевые).

### 4. ARPPU (Average Revenue Per Paying User)
Средняя выручка среди пользователей, совершивших хотя бы одну покупку.

### 5. AOV (Average Order Value)
GMV / Количество заказов. Считается глобально.

### 6. Количество заказов на пользователя
Среднее число заказов, агрегированное по пользователям.

### 7. Доля пользователей с ошибкой оплаты
Пользователи, у которых хотя бы раз возникла ошибка оплаты / Все пользователи.

### 8. Время до первой покупки (среди купивших)
Среднее время от регистрации до первой покупки, только для пользователей, совершивших покупку.

### 9. Refund Rate
Количество возвратов / Количество заказов. Считается глобально. Каждый заказ — либо возвращён, либо нет.

### 10. RPM (Revenue Per Mille)
Рекламная выручка / Количество показов × 1000.

### 11. Доля активных пользователей (DAU/MAU)
Пользователи с ≥1 сессией за день / Все зарегистрированные пользователи.

### 12. GMV per user
Сумма стоимости всех заказов пользователя. Агрегируется как per-user mean.

### 13. Средний рейтинг отзыва (среди оставивших отзыв)
Средний балл отзыва, только для пользователей, написавших отзыв.

### 14. Число сессий на пользователя
Среднее число сессий за период, по пользователям.

### 15. Стоимость доставки на заказ
Общая стоимость доставки / Количество заказов.

### 16. Конверсия в подписку среди пользователей, увидевших paywall
Доля пользователей, оформивших подписку, среди тех, кто увидел paywall.

### 17. Retention Day 7
Доля пользователей, вернувшихся на 7-й день после регистрации.

### 18. LTV (Lifetime Value) per user
Кумулятивная выручка за 90 дней на пользователя. Агрегируется как per-user mean.

## Ответы

| # | Тип | Подход | Главный риск |
|---|---|---|---|
| 1 | Долей (Proportion) | z-test / chi-square / bootstrap | Treatment-affected exposure (число показов может зависеть от эксперимента) |
| 2 | Proportion | z-test для долей | Минимальный — стандартный случай |
| 3 | Per-user | t-test / Welch (bootstrap при heavy tail) | Heavy tail (выбросы в revenue) |
| 4 | Conditional | Двухэтапная процедура / bootstrap с коррекцией | Selection bias — состав «платящих» зависит от treatment |
| 5 | Ratio | Delta-method / bootstrap отношения | Нестабильный знаменатель (число заказов случайно) |
| 6 | Per-user | t-test / Welch | Умеренный heavy tail при большом разбросе |
| 7 | Proportion | z-test для долей | Минимальный |
| 8 | Conditional | Bootstrap с коррекцией / регрессия | Selection bias — купившие в тесте и контроле — разные популяции |
| 9 | Долей (Proportion) | z-test / chi-square / bootstrap | Минимальный — каждый заказ либо возвращён, либо нет |
| 10 | Ratio | Delta-method / bootstrap отношения | Нестабильный знаменатель + heavy tail в revenue |
| 11 | Proportion | z-test для долей | Минимальный |
| 12 | Per-user | t-test / Welch (bootstrap при heavy tail) | Heavy tail (GMV) |
| 13 | Conditional | Bootstrap с коррекцией | Selection bias — оставившие отзыв ≠ все пользователи |
| 14 | Per-user | t-test / Welch | Умеренный |
| 15 | Ratio | Delta-method / bootstrap отношения | Нестабильный знаменатель |
| 16 | Conditional | Двухэтапная процедура | Selection bias — увидевшие paywall зависят от treatment |
| 17 | Proportion | z-test для долей | Минимальный |
| 18 | Per-user | Bootstrap (рекомендуется) | Heavy tail + правый скос (LTV) |
